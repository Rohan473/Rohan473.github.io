Token Budgeting · MCP · RAG

Persistent Research Memory

A memory layer that makes long, deep research sessions token-efficient. It stores the settings, configurations, and prior results you'd otherwise re-explain every single time, then compiles just the relevant slice back into the prompt — so the context budget goes to advanced reasoning instead of re-establishing what's already known.

  • Token budgeting at its core: a salience-scored memory store and a budget-aware context compiler assemble the most useful memories within a fixed token limit. Frequently-reused settings and configs are saved once and recalled cheaply instead of being re-derived each session — freeing context for deeper reasoning.
  • Hybrid retrieval fuses semantic vector search (Qdrant + sentence-transformer embeddings) with symbolic graph traversal over a knowledge graph of past experiments.
  • Exposed over a Model Context Protocol (MCP) server, so assistants like Claude and ChatGPT can tap the memory through a standard tool interface.
  • Production-shaped: builds its graph automatically from chat exports (16 sessions → ~165 nodes, 1,295 edges), served by a FastAPI app, Dockerized with tests and GitHub Actions CI — demonstrated on quant research but built as a generic framework.

Full-stack · OCR

ScribeAI — Handwritten Notes Digitizer

A full-stack web app that turns photos of handwritten notes — including mixed English and Hindi — into searchable, editable, organized digital notes. Designed and built end-to-end, solo.

  • Handwriting recognition via the Mistral OCR API, with multi-language support across English and Devanagari (Hindi) script.
  • A real notes workspace: drag-and-drop uploader and a split-pane editor — original image on one side, editable transcription on the other.
  • Folder organization, full-text search across every note, and searchable PDF export (an invisible text layer over the original image via ReportLab).
  • Built end-to-end: FastAPI backend, React frontend, and a MongoDB data model.

Classification · Explainability

Employee Attrition Risk Scoring System

An end-to-end pipeline that predicts which employees are at risk of leaving and produces an interpretable risk score for each one — packaged in an interactive dashboard so non-technical stakeholders can explore it.

  • Benchmarked Logistic Regression, Random Forest, Gradient Boosting, and XGBoost to pick the strongest performer.
  • Tackled class imbalance with SMOTE and validated honestly using stratified k-fold cross-validation.
  • Used SHAP to surface the drivers behind each prediction, turning a black box into something HR teams can reason about.
  • Shipped an interactive Streamlit dashboard for live risk scoring and what-if exploration.

Regression · Optimization

Restaurant Profit Forecasting & Channel-Mix Optimizer

A decision-support tool that forecasts profit and then recommends how to allocate effort across sales channels for the best return — pairing predictive modeling with constrained optimization.

  • Built regression models plus quantile regression to forecast profit with realistic uncertainty bands, not just point estimates.
  • Framed channel allocation as a constrained problem and solved it with SLSQP optimization to recommend an optimal mix.
  • Caught and corrected a data-leakage bug — an R² near 1.0 traced to target-correlated features — and rebuilt the pipeline to report trustworthy performance.
